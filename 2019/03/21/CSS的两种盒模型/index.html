CSS的两种盒模型

什么是盒模型

  • 把HTML页面中的元素看作是一个矩形的盒子，也就是一个盛装内容的容器。

  • 每个矩形都由元素的内容(content)，内边距(padding)，边框(border)和外边距(margin)组成。

  • 网页其实就是多个盒子嵌套排列的结果

  • 内边距出现在内容区域的周围，当给元素添加背景色或背景图像时，该元素的背景色或背景图像也将出现在内边距中

  • 外边距是该元素与相邻元素之间的距离

  • 如果给元素定义边框属性，边框将出现在内边距和外边距之间

  • 虽然盒子模型拥有内边距，边框，外边框，宽和高这些基本属性，但是并不是每个元素都必须定义这些属性。

两种盒模型

因为浏览器兼容性不一样，出现了两种盒模型：

  • W3C盒子模型(标准盒模型)

  • IE盒子模型(怪异盒模型)

  • 区别：计算宽度和高度的不同。

    • 在标准的盒模型中，width/height指content部分的宽度/高度
    • 在IE怪异盒模型中，width/height表示content+padding+border这三个部分的宽度/高度

  • 在工作中使用的是标准盒模型

盒模型的切换方式

  • box-sizing: content-box 标准盒模型
  • box-sizing: border-box 怪异盒模型

box-sizing的默认属性是content-box
